FORMULA USED:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

x = 575 - 175 * (distance / 100)
y = 82 + 141 * (distance / 100)
width = 46 + 137 * (distance / 100)
height = 52 + 80 * (distance / 100)

Where:
  - 575 to 400 = 175 pixel horizontal movement (left)
  - 82 to 223 = 141 pixel vertical movement (down)
  - 46 to 183 = 137 pixel width expansion
  - 52 to 132 = 80 pixel height expansion

All four properties use:
  - Duration: 300ms per frame update
  - Easing: InOutQuad (smooth acceleration/deceleration)
  - Continuous update as distance changes


EXPECTED BEHAVIOR WHEN YOU RUN:

1. App starts → car appears small and high (farthest position)
2. For 4 seconds → car smoothly slides left, down, and grows
3. Car reaches maximum size and lowest position (approaching)
4. Car pauses for 0.5 seconds at closest position
5. For 4 seconds → car smoothly slides right, up, and shrinks
6. Car returns to starting position (farthest)
7. Cycle repeats infinitely

This will loop continuously, demonstrating:
  ✓ Smooth position transitions (300ms easing)
  ✓ Correct coordinate progression
  ✓ Size scaling relative to perceived distance
  ✓ Realistic perspective (larger = closer, lower = closer)
