# BatteryMonitor Project: ThreadX Functions Overview

This project uses the ThreadX RTOS to manage concurrency and timing for STM32-based battery monitoring and motor/servo control. Below is an overview of the key ThreadX functions and concepts used in the codebase.

# ThreadX Functions in BatteryMonitor Project

This document explains the ThreadX RTOS functions and concepts used in the BatteryMonitor project, with details on function prototypes, input parameters, return values, and usage context.

---

## Key ThreadX Functions Used

### 1. `tx_thread_sleep()`
- **Prototype:**
  ```c
  UINT tx_thread_sleep(ULONG timer_ticks);
  ```
- **Inputs:**
  - `timer_ticks`: Number of ThreadX ticks to sleep (yield CPU to other threads). Tick duration is set in the RTOS config (often 10ms per tick).
- **Returns:**
  - `TX_SUCCESS` (0) if successful, or an error code if the call fails.
- **Usage in Project:**
  - Used for timing delays (e.g., waiting for I2C bus stabilization, pacing LED blinking, and providing delays between motor/servo actions).
  - Example: `tx_thread_sleep(100); // Sleep for 1 second if tick = 10ms`

### 2. `MX_ThreadX_Init()`
- **Prototype:**
  ```c
  VOID MX_ThreadX_Init(VOID);
  ```
- **Inputs:**
  - None (void)
- **Returns:**
  - None (void)
- **Usage in Project:**
  - Called in `main()` after hardware initialization to start the ThreadX kernel and create application threads.
  - This function is typically auto-generated by STM32CubeMX and sets up thread stacks, priorities, and entry points.

### 3. Thread Entry Function (`Battery_Thread_Entry`)
- **Prototype:**
  ```c
  void Battery_Thread_Entry(ULONG thread_input);
  ```
- **Inputs:**
  - `thread_input`: Optional user data passed to the thread (unused in this project, usually 0).
- **Returns:**
  - None (void)
- **Usage in Project:**
  - Main application logic: I2C device scanning, PCA9685 initialization, periodic battery monitoring, and test routines for motors/servos.
  - Registered as a thread entry point in the ThreadX initialization code (see `app_threadx.c`).

---

## ThreadX Concepts in Use

- **Single Thread Model:**
  - The project currently uses a single main thread (`Battery_Thread_Entry`) for all application logic. ThreadX is capable of running multiple threads, but only one is used here for simplicity.

- **Cooperative Multitasking:**
  - By calling `tx_thread_sleep()`, the thread yields control, allowing the RTOS to manage timing and (if present) other threads.

- **Deterministic Timing:**
  - ThreadX tick-based delays ensure consistent timing for hardware operations, such as I2C communication and PWM signal changes.

## Example Code Snippet
```c
void Battery_Thread_Entry(ULONG thread_input) {
    // ...
    tx_thread_sleep(100); // Wait 1 second
    // ...
    while(1) {
        // Periodic work
        tx_thread_sleep(200); // 2 seconds
    }
}
```
