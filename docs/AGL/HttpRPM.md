# Http server for RPM packages

This document describes the process to create the http server for RPM packages generated by AGL

1. Generate the metadata for dnf

This is done in the Target PC inside the AGL enviroment

```bash
bitbake package-index
```

2. Go to the RPM directory

The normal path is <build-dir>/tmp/deploy/rpm

```bash
cd <build-dir>/tmp/deploy/rpm
```

3. Host the http server

```bash
python3 -m http.server 8000
```

4. Create a dnf config file

In the Target PC go to **/etc/yum.repos.d**.
Now create a file with this configuration
Put your HostPC-IP in the baseurl

```ini
[localdnf]
name=local-rpms
baseurl=http://<Your-HostPC-IP>:8000/
enabled=1
gpgcheck=0
```

5. Update dnf cache

Any time the rpm files are altered, you need to update the cache, with this

```bash
dnf makecache
```

6. Check if it's alright

Now it must be possible to list all the rpm files without errors, with this

```bash
dnf list
```
